day01

面向对象编程介绍

面向过程:

  • 优点:性能高,性能比面对对象高,适合硬件联系很紧密的东西,
  • 缺点:没有面向对象易维护,易复用,易扩展
  • eg:单片机

面向对象:

  • 优点:易扩展,易维护,易复用,代码具有灵活,可复用,容易维护和开发,适合大型开发项目
  • 缺点:没有面向过程性能高
  • 特点:封装性,继承性,多态性

总结

  • 面向对象和面向过程都是编程思想
  • 面向过程是分析出解决问题所需要的步骤,然后用函数一步一步实现. (c++)
  • 面向对象把事物分析成一个个对象,利用对象功能来划分功能(Java)
  • JS 既可以面对对象编程也可以面向过程编程.

类和对象的区别

  • 类(class):类抽象了对象的公共部分,它泛指一大类 例如:苹果 明星
  • 对象:现实中万物皆对象,对象是一个具体的事物,看得见摸得找 例如 手机的属性 手机屏幕多大 CPU 手机的方法 打游戏 看视频

call() apply() bind()总结

​ 都可以改变函数内部的 this 指向

区别点

  1. call 和 apply 会调用函数,并且改变函数内部的 this.
  2. call 和 apply 传递的参数不一样, call 传递参数 aru1, aru2 形式 apply 必须以数组形式[arg]
  3. bind 不会调用函数,可以改变函数内部 this 的指向

主要应用场景

  1. call 经常做继承
  2. apply 经常跟数组有关系,比如借助与数学对象实现数组最大值和最小值
  3. bind 不调用函数,但是还想改变 this 指向.比如改变定时器内部 this 的指向

严格模式

什么是严格模式?

  • JavaScript 除了提供正常的模式外,还提供了严格模式(strict mode).ES5 的严格模式采用了具有限制性

  • JavaScript 变体的一种方式,即在严格的条件下运行 JS 代码

开启严格模式

  • 同样的，要给某个函数开启严格模式，得把 "use strict"; (或 'use strict'; )声明一字不漏地放在函数体所有语句之前

严格模式下 this 指向问题

  1. 以前在全局作用域函数中的 this 还是指向 window
  2. 严格模式下全局作用域中的 this 是 undefined
  3. 以前构造函数式不加 new 也可以调用,当做普通函数,this 指向全局对象
  4. 严格模式下,如果 构造函数不加 new 调用,this 指向 undefined 如果给它赋值会报错
  5. new 实例化的构造函数指向创建的对象实例
  6. 定时器 this 还是指向 window
  7. 事件,对象还是指向调用者

函数的变化

  1. 函数不能有重名的参数

  2. 函数必须声明在顶层.新版本的 JavaScript 会引入”块级作用域”(ES6 已引入).为了与新版本接轨,不允许在非函数的代码内声明还是

创建正则表达式

  • 正则表达式（ Regular Expression ）是用于匹配字符串中字符组合的模式。在 JavaScript 中，正则表达式也是对象。
  • 特点:
    • 灵活性、逻辑性和功能性非常的强
    • 可以迅速地用极简单的方式达到字符串的复杂控制
  1. 正则表达式的两种创建方式
    1. 通过 new 关键字来创建 var regExp = new RegExp(/123/)
    2. 通过字面量的方式来创建 // var rg = /123/
  2. 检测
    • test 方法用来检测字符串是否符合正则表达式要求的规范
