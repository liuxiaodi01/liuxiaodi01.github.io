工具 nodemon

  • 在编写调试 Node.js 项目的时候,如果修改了项目代码需要频繁的手动 close 和重启,非常繁琐

中间件

中间件的用处:简化代码,在注册了中间件可以共享属性和方法给后面所有的路由

不使用 app.use()注册的中间件叫做局部中间件

定义多个中间件

了解中间件的 5 个使用注意事项

  1. 一定要在路由之前注册中间件
  2. 客户端发送过来的请求,可以连续调用多个中间件进行处理
  3. 执行玩中间件的业务代码之后,不要忘记调用 next()函数
  4. 为了防止代码逻辑混乱,调用 next()函数后不要再写额外代码了
  5. 连续调用多个中间件时,多个中间件之间,共享 req 和 res 对象

为了方便大家理解和记忆中间件的使用,Express 方法把常用的中间件分成了 5 大类,分别是

  1. 应用级别的中间件 绑定到app实例上的叫应用级别中间件

  2. 路由级别的中间件 绑定到router实例上的叫路由级别中间件

  3. 错误级别的中间件

    防止项目异常崩溃问题 错误级别中间件的function的参数(err,req,res,next)

    例外:必须注册在所有路由之后

      cors 跨域资源共享

      1.什么 cors

      由一系列 http 响应头组成，这些 http 响应头决定浏览器是否阻止前端 js 代码获取资源

      如果配置了 cors 相关的 http 的响应头，就跨域解决浏览器端的跨域访问限制

      cors 响应头-Access-Control-Allow-Origin

      res.setHeader(“Access-Control-Allow-Origin”,”http://xiaodi.cn") 这个字段值将只允许来自 xiaodi.cn 的请求

      res.setHeader(“Access-Control-Allow-Origin”,”*“) 允许所有请求

      cors 响应头-Access-Control-Allow-Headers

      cors 默认支持客户端向服务器发送的 9 个请求头

      在这 9 个之外请求头需要额外进行声明

      res.setHeader(“Access-Control-Allow-Headers”,”Content-type”,”X-Custom-Header”) 设置额外的请求体

      cors 响应头-Access-Control-Allow-Methods

      默认情况下,cors 仅支持 get,post,head 请求

      res.setHeader(“Access-Control-Allow-Methods”,”*“) 允许所有的请求方式

      2cors 请求的分类(两大类)
      简单请求
      1. 请求方式:get,post,head 三者之一
      2. HTTP 头部信息不超过上述 9 个字段和无自定义自定义
      预检请求

      1. 请求方式:get,post,head 之外的

      2. HTTP 头部信息包含自定义字段

      3. 向服务器方式了 application/json 格式数据

        在服务器和浏览器正式通信之前,浏览器会先发送OPTION请求进行预检,以获知服务器是否允许该实际请求,所以这一次的OPTION请求被称为"预检请求".服务器成功响应才会发送真正的请求,并返回真正的数据

    简单请求和预检请求的区别

    简单请求的特点:客户端和服务器之间只会发送一次请求

    预检请求的特点:客户端与服务器之间会发生两次请求,OPTION 预检成功,才会发送请求

    jsonp

    • 概念:浏览器通过 scrpit 标签的 src 属性,请求服务器上的数据,同时服务器返回一个函数调用,这种请求就叫做 jsonp

    • 特点

      1. jsonp 不属于真正的 ajax 请求,因为他没有使用 XMLHttpRequest 这个对象
      2. jsonp 仅支持 get 请求,不支持 post 和,put,delete 请求
