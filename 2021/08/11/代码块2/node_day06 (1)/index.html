ES6模块化

提供通用方式实现成员的暴露和引入，适用于服务器端和浏览器端

之前node中模块化的相关操作

只是适用于服务器端

  • 暴露(导出)
    • module.exports:暴露成员，最终以module.exports所指向的对象为准
    • exports:
    • global:全局对象，相当于html中的window,可以在global挂载成员，不同模块之间通用
    • 全部暴露
      • module.exprots = {成员….}
    • 按需暴露
      • module.exprots.成员 = 业务代码
  • 引入
    • require(核心模块名称|第三方模块|自定义模块)

ES6模块化规范

  • 前期准备：
    • 创建一个项目，初始化package.json文件
    • 在package.json文件中 设置 type:module
  • 默认导出和默认导入
    • 使用场景：当前模块中只有一个成员对象需要被导出，这个时候应该使用默认导出
    • 代码：
      • 导出代码：export default 成员
      • 导入代码：import 变量 from ‘模块路径’
    • 细节：
      • 一个模块只能有一个default
      • 引入模块的时候记得添加扩展名
      • 导入的时候接收的变量要符合变量的命名规则
  • 按需导出和按需导入
    • 使用场景：当前模块中有多个成员需要被导出
    • 代码：
      • 导出代码：export 你想暴露的成员
      • 导入代码：import {你想 导入的成员} from ‘模块路径’
    • 细节：
      • 导入的成员的名称必需和导入的成员名称一致
      • 我们可以对导入的变量 通过 as 关键字进行重命令，只是需要注意重命名之后，我们需要使用的是重定名之后的变量，如果使用之前，就报错 import {age as myage } from ‘’
      • 默认导出和按需导出可以一起使用，注意默认导出只有一个，注意在导入的时候，先导入默认成员，再导入按需的成员
  • 导出之后，直接导入执行
    • 本质是因为你导入一个模块，就会编译这个模块，意味着会执行模块中的代码，所以让我们感觉是直接执行
    • 特点：引入之后不用接收
    • 使用场景：引入样式

promise

作用：可以用于解决回调地狱

1.它是一个构造函数，所以我们要使用promise得先创建一个promise对象

  • let p = new Promise(参数)

2.在Pormise构造函数的原型上有.then 和.catch两个方法

3.使用方式

  • p.then(成功的回调函数，失败的回调函数)
  • p.then(成功的回调函数).catch(失败的回调函数)

使用then-fs异步方式读取文件

使用promise方式按顺序读取文件内容

  • 如果在then中返回一个promise,那么就可以继续调用promise的then方法

捕获promise操作过程中的错误

可以添加一个.catch(err=>{})

调用.catch之后，返回的也是一个promise对象

特点：

  • 只要代码出错，就会自动的触发.catch
  • 在默认情况下，代码出错了，就会触发.catch，就不会再执行后续的代码，如果你希望出错不影响后续代码的执行，可以将catch提前

两个方法的补充

这两个方法是通过Promise进行调用

Promise.all:

执行多个异步操作，只有多个异步都执行完毕，才去执行.then方法

Promise.race:

传入多个异步操作，只要其中一个异步操作执行完毕，就会立刻执行.then方法

填写一个基于promise文件读取操作

用户可以传入一个文件路径，方法负责返回一个promise对象

调用方法，获取到promise对象之后，可以.then和.catch

当有多个文件需要进行读取的时候，避免回调地狱

async和await

重点内容，项目中必用

EventLoop

进行任务队列的监听，如果在任务列表中有任务需要执行，就会依次执行

1.js是单一线程，会造成线程死锁，同步会阻塞后面代码的执行

2.js将非耗时任务在主线程来执行，耗时任务就让委托给宿主环境来执行

3.宿主环境有：浏览器 nodejs

4.如果在主线程中有异步操作，就会将异步操作委托给宿主环境来执行，宿主环境中执行完异步操作之后，会将回调函数添加到任务队列中，主线程如果执行完所有非耗时任务，就会自动的扫描任务队列中是否有任务需要执行，如果有，就会依次执行
